# MathMaster FBLA Website Design Script (<= 7 minutes)

Estimated spoken time: **6:05-6:35**

## [0:00-0:45] Hook + Framing (Ayaan)
Good afternoon, judges.

Let me start with a quick question: if students have so many learning apps, why do so many still feel behind in math?

From our interviews, the issue was not effort. It was fit. Students told us they were overwhelmed by disconnected tools, inconsistent support, and practice that did not adapt to their level.

So we built **MathMaster**: a student-built, student-centered learning hub aligned to this year’s topic, **Design to Learn**, to make math more engaging, collaborative, and accessible.

## [0:45-1:55] Planning, Development, and Implementation (Malhar)
We built MathMaster in three phases: planning, development, and implementation.

In planning, we collected feedback from students, tutors, and teachers, then converted that into measurable product goals: improve clarity, increase engagement, and reduce drop-off in study sessions.

In development, we benchmarked proven systems and standards: Khan Academy for structured progression, IXL for targeted practice patterns, and WCAG principles for accessibility-driven design.

In implementation, we used sprint cycles with a Gantt chart and role-based ownership. Every week, we reviewed bugs, tested usability, and shipped improvements. That process gave us a complete planning-to-launch workflow with evidence behind major design choices.

## [1:55-2:55] UX Design, Accessibility, and Visual Consistency (Khush)
Our UX philosophy is simple: students should understand what to do in seconds.

We used a consistent visual system across pages: repeatable layouts, clear contrast, readable typography, and predictable navigation patterns.

Accessibility is built into the product, not added as an afterthought. Users can adjust text size, turn on high-contrast mode, enable dyslexia-friendly font options, reduce motion, and navigate with keyboard-friendly controls.

These choices make the experience inclusive while keeping the interface polished and cohesive.

## [2:55-5:25] Live Walkthrough of Required Elements + Value-Add (Team)
### [2:55-3:35] Home Page and First-Click Experience (Malhar)
Starting on the home page, users immediately see practical next actions instead of decorative clutter.

Students can quickly access learning tools, AI support, and productivity features that help them start work right away.

This reduces cognitive load and creates a faster path from landing on the site to active learning.

### [3:35-4:10] Required Element 1: Schedule Page (Ayaan)
Next is our **Schedule** page, which satisfies the first required element.

Students can find live tutoring sessions and group study opportunities by subject and time.

This supports real peer-to-peer learning and gives students a structured way to ask for help before they fall behind.

### [4:10-4:45] Required Element 2: Student Dashboard (Khush)
Our **Dashboard** satisfies the second required element.

Students can track completed work, monitor progress, and see suggested next steps.

We designed this page to convert progress from a static number into actionable momentum.

### [4:45-5:25] Required Element 3: Resources Section (Malhar)
Our **Resources/Learn** section satisfies the third required element.

It includes interactive lessons, guided practice, quizzes, videos, and downloadable study materials.

Beyond baseline requirements, we add tools like graphing and quick calculation support so students can move from concept learning to problem solving in one flow.

## [5:25-6:05] Functionality, Compatibility, Interactivity, and Metrics (Ayaan)
From a technical standpoint, MathMaster is designed for reliability across **desktop, tablet, and mobile**.

Interactive components such as quizzes, calculators, and guided learning actions are tested to run smoothly and reduce user friction.

To measure success, we track key website and learning metrics: session duration, return rate, completion rate, topic-level accuracy growth, tutoring booking conversion, and drop-off points in user flows.

Using those metrics, we continuously refine both user experience and learning outcomes.

## [6:05-6:35] Sources, Credibility, and Close (Khush -> Ayaan)
All major claims and design decisions are supported by professionally credible references, and all outside resources are cited.

(Ayaan) MathMaster is more than a website mockup. It is a student-built learning ecosystem that makes math less isolating, more interactive, and more achievable.
